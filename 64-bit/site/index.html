Why this book exists

Learning C is easy. Using it to build an operating system is not.

Most C books assume an operating system already exists — they begin at main(), print with printf(), and allocate with malloc(). Kernel development removes all of that. There is no standard library, no allocator, no debugger waiting in the background. The software you write has to create those services itself.

This book bridges that gap. Every chapter takes a feature of C — pointers, structures, bit manipulation, inline assembly, linking — and shows it doing real work inside a kernel you can compile and boot. The language is the same one you already know; the environment is what changes. By the end, opening an unfamiliar kernel source file should feel like a place you know how to explore, not a wall.

It assumes you know the fundamentals of C. It does not assume any prior operating-systems or assembly experience — both are introduced only where the code makes them necessary.

How to use it

Keep the repository open. The book is the guide; the kernel is the lab.

Read a chapter, then open the file it points to. Compile the kernel, boot it under an emulator, change one line, and boot it again. The chapters expect you to work alongside the tutorial rather than read straight through — understanding here grows from observation as much as from reading.

# build the minimal screen kernel from Appendix B $ cd 64-bit/03_screen_64/src $ make $ qemu-system-x86_64 -kernel kernel # you should see, in the top-left corner: Hello, world! # then change one thing, and boot again $ $EDITOR main.c // monitor_write("...your text...");
